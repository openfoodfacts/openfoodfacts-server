This file describes fields used in Open Food Facts data exports.

See https://world.openfoodfacts.org/data for downloads.

Primary documentation sources used for this file:
- OpenAPI product schema: docs/api/ref/api.yaml and docs/api/ref/schemas/product*.yaml
- openfoodfacts-exports service: https://github.com/openfoodfacts/openfoodfacts-exports

The file encoding is UTF-8.
CSV files use <tab> as field separator.

General field conventions:
- fields ending with _t are UNIX timestamps (seconds since 1970-01-01)
- fields ending with _datetime are ISO 8601 timestamps (yyyy-mm-ddThh:mm:ssZ)
- fields ending with _tags are normalized/tagified values, generally for search/filtering
- fields ending with _hierarchy are non-lossy hierarchical tag entries for display/editing
- fields ending with _lc indicate the language used for the last edit of the corresponding field
- fields ending with _<language_code> are localized values (e.g. ingredients_text_fr)
- nutriments use suffixes such as _100g, _serving, _value, _unit and prepared_* variants

======================================================================
CSV EXPORTS
======================================================================

There are multiple CSV/TSV exports with different field sets.

A) Full historical TSV export (server-side export_database.pl)
- Starts from configured export groups (ProductOpener::Config_off import_export_fields_groups)
- Adds derived columns such as:
  - url (after code)
  - *_datetime for *_t fields
  - *_tags for selected tag fields
  - localized taxonomy values for selected fields
  - first_packaging_code_geo (from first emb code)
  - main_category and localized main_category_<lc>
  - selected image URLs (front/ingredients/nutrition)
  - nutrient columns as <nutrient>_100g
- Nutrient list follows nutrients taxonomy/export logic and evolves over time.

B) Generic CSV export endpoint (ProductOpener::Export::export_csv)
- Exports populated fields (or explicit requested fields)
- Supports optional computed fields and canonicalized tags
- Supports optional image local paths
- Supports dynamic columns for:
  - language-specific text fields
  - nutrition fields
  - packaging component fields (e.g. packaging_1_shape)

C) openfoodfacts-exports mobile TSV
Source: openfoodfacts_exports/exports/csv/mobile.py
Columns:
- code
- product_name
- quantity
- brands
- nutrition_grade_fr (alias of nutriscore_grade)
- nova_group
- environmental_score_grade

D) openfoodfacts-exports parquet datasets
Source: openfoodfacts_exports/exports/parquet/common.py and food.py
These files define current normalized field sets used for modern exports.
