get_inherited_property_from_categories_tags ($product_ref, $property) {

Iterating from the most specific category, try to get a property for a tag by exploring the taxonomy (using parents).

Parameters

$product_ref - the product reference

$property - the property - string

Return

$property_value

The property value if found.

$matching_category_id

The matching category id if we found a property value.

get_all_tags_having_property ($product_ref, $tagtype, $prop_name)

For each tag of a given field ($tagtype, can be "labels" or "categories", for example), and a given property ($prop_name, without last column (:). Can be "incompatible_with:en", for example), return a hash of tagid <-> property_value remark: this DOES NOT handle property inheritance

Return

A hash, where keys are tagid and values are property_value

Example, get_all_tags_having_property($product_ref, "labels", "incompatible_with:en")

FUNCTIONS

add_tag

Adds a tag to a specified tag type in the product reference if it does not already exist.

Arguments

$product_ref

A hash reference to the product data.

$tagtype

The type of the tag (e.g. categories, labels, allergens).

$tagid

The ID of the tag to add.

Return value

Returns 1 if the tag was added, 0 if it already existed.

add_tags_to_field ($product_ref, $tag_lc, $field, $additional_fields)

Add a comma separated list of values in the $lc language to a taxonomy field.

Note: this function is only for fields that are generated (e.g. misc tags and data quality tags). It is not used for input tags fields. Instead set_field_input_tags_for_source() is used.

Parameters

$product_ref

Reference to the product hash.

$tag_lc

The language code of the tags to add.

$field

The field to which the tags should be added.

$additional_fields

A comma separated list of tags to add to the field.

set_field_input_tags_for_source ($product_ref, $tag_lc, $field, $source, $input_tags)

New function to set the input tags for a field and a source. (e.g. categories for the manufacturer source)

Note that there is special logic for the allergens field, to split traces from allergens if the user entered them together in the allergens field. Those traces might be overridden by the client if it also calls set_field_input_tags_for_source with empty traces after setting allergens.

Parameters

$product_ref

Reference to the product hash

$tag_lc

The language code of the input tags

$field

The field for which we want to set the input tags (e.g. categories, labels, allergens, etc.)

$source

The source of the input tags (e.g. manufacturer, user, etc.)

$input_tags

The input tags to set for the field and source

$add

Optional flag to indicate whether to add the input tags to existing tags (default is 0, which replaces existing tags)

generate_field_tags_from_all_sources ($product_ref, $tagtype)

This function gathers all the input tags for a field from all sources, and generates the final tags for the field, including parents for taxonomy fields.

compute_field_tags ($product_ref, $tag_lc, $field)

Generate the tags hierarchy from the comma separated list of $field with default language $tag_lc

This function was used primarily before we refactored tags with tags_sources (schema version < 1005).

It is still used to upgrade old products to newer schema (see ProductSchemaChanges.pm)
